## 1. 环境准备

```bash
# 进入项目（按你本机路径改）
cd /Users/chenxiaofei/开发/项目/亦失/grok-build

# Rust 工具链（项目有 rust-toolchain.toml，会自动装）
export PATH="$HOME/.cargo/bin:$PATH"
rustc --version
cargo --version
```

- **macOS**：本机直接编译即可。  
- **Windows**：在 Windows 上装 Rust + MSVC 工具链，或靠 GitHub Actions 出 `yis.exe`。  
- 编译产物目录：`target/`（**很大，可随时删**，见第 7 节）。

## 2. 日常开发命令

### 2.1 开发模式运行（常用）

```bash
export PATH="$HOME/.cargo/bin:$PATH"
cd /Users/chenxiaofei/开发/项目/亦失/grok-build

# 编译并启动 TUI
cargo run -p xai-yis-pager-bin

# 已编过时直接跑
./target/debug/yis
```

### 2.2 只检查能否编译（快）

```bash
cargo check -p xai-yis-pager-bin
cargo check -p xai-yis-shell --lib
```

### 2.3 Debug 编译

```bash
cargo build -p xai-yis-pager-bin
# 产物: target/debug/yis
```

### 2.4 Release 编译（给自己用、体积更小）

```bash
cargo build -p xai-yis-pager-bin --release
# 产物: target/release/yis
strip target/release/yis   # 可选，再压体积

# 发行向（更慢、优化更狠）
cargo build -p xai-yis-pager-bin --profile release-dist
# 产物: target/release-dist/yis
```

### 2.5 本机打当前架构发布包

```bash
./scripts/build-release-local.sh
# 输出到 dist/yis-darwin-arm64 等
```

## 3. 模型配置（本地 BYOK，不连 grok.com）

```bash
# 交互配置（推荐）
./target/debug/yis models setup
# 或 release 二进制
./target/release/yis models setup

# 非交互
yis models add --preset deepseek --api-key sk-你的密钥
yis models add --preset deepseek --api-key sk-... --model deepseek-reasoner

# 列表 / 预设
yis models
yis models presets
```

配置文件：`~/.yis/config.toml`  

```toml
[models]
default = "deepseek-chat"

[model.deepseek-chat]
model = "deepseek-chat"
name = "DeepSeek"
base_url = "https://api.deepseek.com/v1"
api_backend = "chat_completions"
api_key = "sk-..."
context_window = 128000
```

TUI 内：

| 命令 | 作用 |
|------|------|
| `/model-add` | 模型管理（厂商 / Key） |
| `/model` 或 `Ctrl+M` | 切换模型 |
| `/language` | 中英切换 |

预设 id：`deepseek` `bailian` `zhipu` `kimi` `minimax` `mimo` `openai-compatible` `anthropic-compatible` `custom`

改预设代码：`crates/codegen/xai-yis-shell/src/model_presets.rs`  
TUI 弹窗厂商列表：`crates/codegen/xai-yis-pager/src/views/model_manager.rs`  
CLI 向导：`crates/codegen/xai-yis-pager/src/model_setup_cmd.rs`

## 4. 本地隐私模式（你改过的核心行为）

| 点 | 说明 |
|----|------|
| Release 构建 | **始终本地模式**，不连 grok.com 登录/上传 |
| Debug 构建 | 默认本地；可用 `YIS_LOCAL_MODE=0` 临时恢复上游云（仅开发） |
| 真源 | `crates/codegen/xai-yis-env/src/lib.rs` → `is_local_mode()` |

```bash
# 强制本地
YIS_OFFLINE=1 ./target/debug/yis

# 仅 Debug：尝试关本地（Release 无效）
YIS_LOCAL_MODE=0 ./target/debug/yis
```

`yis login` 在本地模式下会直接提示不可用。

## 6. Git 与发版（macOS + Windows）

### 6.1 远程仓库

```bash
# 查看
git remote -v

# 绑到 yis-Cli（若还没有）
git remote add yis https://github.com/481617494/yis-Cli.git
# 或改 origin：
# git remote set-url origin https://github.com/481617494/yis-Cli.git
```

### 6.2 提交并推送

```bash
git status
git add -A
git commit -m "说明这次改了什么"
git push yis main          # 或 origin main
```

### 6.3 打 tag 自动发布（GitHub Actions）

```bash
git tag v0.1.0
git push yis v0.1.0
```

成功后用户安装：

```bash
# macOS
curl -fsSL https://github.com/481617494/yis-Cli/releases/latest/download/install.sh | bash

# Windows PowerShell
irm https://github.com/481617494/yis-Cli/releases/latest/download/install.ps1 | iex
```

Release 资产：

| 文件 | 平台 |
|------|------|
| `yis-darwin-arm64` | Apple Silicon |
| `yis-darwin-x64` | Intel Mac |
| `yis-windows-x64.exe` | Windows 64 位 |
| `install.sh` / `install.ps1` | 安装脚本 |
| `checksums.txt` | 校验 |

工作流文件：`.github/workflows/release.yml`  
在 GitHub → **Actions** 看构建日志。

### 6.4 只改安装脚本 / 文档

不需要重新编译二进制时，改完 `scripts/` 或 `docs/` 推 `main` 即可；  
**安装脚本进 Release 是随 tag 打的**，改完 install 脚本后需要再打新 tag 才会更新 `releases/latest/download/install.sh`。

## 7. `target/` 目录：可以删吗？

### 结论：**可以，而且建议磁盘紧时删。**

| 问题 | 答案 |
|------|------|
| 是什么？ | Rust/Cargo 编译缓存 + 中间文件 + 生成的 `yis` |
| 会不会丢源码？ | **不会**。源码在 `crates/` 等目录 |
| 会不会进 Git？ | 一般 **不该提交**；删了不影响仓库 |
| 删了之后？ | 下次 `cargo build` / `cargo run` 会**重新编译**（第一次较慢） |

### 怎么删

```bash
cd /Users/chenxiaofei/开发/项目/亦失/grok-build

# 推荐：官方清理（删整个 target）
cargo clean

# 或手动
rm -rf target

# 若有本地 dist 发布目录，也可删（可再生成）
rm -rf dist
```

查看占用：

```bash
du -sh target
df -h .
```

### 什么时候删

- 磁盘不够（编译报 `No space left on device`）  
- 长时间不用，想腾空间  
- 怀疑缓存坏了、想干净重编  

### 什么时候别慌

- 删 `target` **不会**删 `~/.yis/config.toml`（你的模型 Key 还在）  
- **不会**删 Git 历史  
- 只是下次编译更久

## 8. 常用排错

| 现象 | 处理 |
|------|------|
| `No space left on device` | `cargo clean`，腾出几十 GB 再编 |
| `yis: command not found` | 用全路径 `./target/debug/yis`，或把 `~/.local/bin` 加 PATH |
| 对话失败 / 401 | `yis models` 看配置；检查 `api_key`、`base_url`；`/model-add` |
| 还在要登录 grok | 确认用的是本仓库编的 `yis`，不是官方 xAI 安装的 |
| Actions 失败：`protoc` not found | CI 需安装 protoc（workflow 已用 `arduino/setup-protoc`）。本机：`brew install protobuf` 或装 `dotslash` 后用仓库 `bin/protoc` |
| Actions 失败 | 打开 GitHub Actions 日志；常见超时/磁盘，可重跑 job |
| 改完代码没效果 | 确认跑的是新编译的二进制，不是旧的 `~/.local/bin/yis` |

## 9. 建议工作流（以后改功能）

```text
1. 改代码（见第 5 节表）
2. cargo check -p xai-yis-pager-bin
3. cargo run -p xai-yis-pager-bin   # 或 ./target/debug/yis
4. 测：yis models / 对话 / 相关斜杠命令
5. git commit + push main
6. 需要给别人装新版本时：git tag vX.Y.Z && git push … vX.Y.Z
7. 磁盘紧了：cargo clean
```
