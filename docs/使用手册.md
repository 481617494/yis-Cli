# Yis Cli 使用手册

Yis Cli 是终端里的 **AI 编程助手**（本仓库二次发行版）。  
默认 **本地安全模式**：不登录 grok.com，不上传对话到 xAI 云；只请求你自己配置的模型厂商 API。

| 文档 | 用途 |
|------|------|
| **本文** | 安装、配置、日常使用、常见问题 |
| [本地开发与运维.md](./本地开发与运维.md) | 编译、改代码、发版、清理 target |
| [YIS_CLI.md](../YIS_CLI.md) | 产品特性与隐私说明 |

仓库：https://github.com/481617494/yis-Cli

---

## 1. 它能做什么？

在项目目录打开终端，运行 `yis` 后可以：

- 用自然语言描述任务（改代码、查 bug、写脚本、解释仓库等）
- 读写文件、执行终端命令（经你确认的安全策略）
- 管理多轮会话、压缩上下文、切换模型
- **不需要** xAI / Grok 账号

出站网络主要是：**你配置的模型 API**（如 DeepSeek、百炼等）。

## 2. 安装

### 2.1 一键安装（有 GitHub Release 之后）

**macOS**

```bash
curl -fsSL https://github.com/481617494/yis-Cli/releases/latest/download/install.sh | bash
```

默认装到 `~/.local/bin/yis`。若提示找不到命令：

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**Windows（PowerShell）**

```powershell
irm https://github.com/481617494/yis-Cli/releases/latest/download/install.ps1 | iex
```

### 2.2 从源码编译（开发机）

```bash
cd /path/to/yis-Cli   # 或本地 grok-build 目录
export PATH="$HOME/.cargo/bin:$PATH"
cargo build -p xai-yis-pager-bin --release
./target/release/yis
```

### 2.3 检查是否装好

```bash
yis --version
# 或
yis models presets
```

## 3. 第一次使用（必做：配置模型）

本地模式 **没有** `yis login`。必须先配置至少一个厂商的 API Key。

### 方式 A：交互配置（推荐）

```bash
yis models setup
```

按提示：选厂商 → 填 API Key → 选模型 → 自动写入 `~/.yis/config.toml`。

### 方式 B：一条命令

```bash
yis models add --preset deepseek --api-key sk-你的密钥
# 指定模型：
yis models add --preset deepseek --api-key sk-... --model deepseek-reasoner
```

### 方式 C：手写配置

编辑 `~/.yis/config.toml`：

```toml
[models]
default = "deepseek-chat"

[model.deepseek-chat]
model = "deepseek-chat"
name = "DeepSeek"
base_url = "https://api.deepseek.com/v1"
api_backend = "chat_completions"
api_key = "sk-你的密钥"
context_window = 128000
```

### 内置厂商预设

| 预设 id | 名称 |
|---------|------|
| `deepseek` | DeepSeek |
| `bailian` | 阿里云百炼 |
| `zhipu` | 智谱 GLM |
| `kimi` | Kimi / Moonshot |
| `minimax` | MiniMax |
| `mimo` | 小米 MIMO |
| `openai-compatible` | 任意 OpenAI 兼容中转 |
| `anthropic-compatible` | Anthropic 兼容 |
| `custom` | 完全自定义 base_url |

查看全部：

```bash
yis models presets
yis models              # 已配置的模型
```

## 4. 启动与基本对话

### 4.1 启动

```bash
# 在任意项目目录下启动（会带上当前目录上下文）
cd ~/你的项目
yis
```

指定模型：

```bash
yis -m deepseek-chat
```

无界面、单次提问（适合脚本）：

```bash
yis -p "用一句话说明这个仓库做什么"
```

### 4.2 界面大致结构

```
┌─────────────────────────────────────┐
│  对话历史（滚动区）                    │
│  … agent 回复 / 工具调用 …            │
├─────────────────────────────────────┤
│  输入框：输入问题，回车发送             │
└─────────────────────────────────────┘
```

- 在底部输入框打字，**Enter** 发送  
- 输入 `/` 打开斜杠命令菜单  
- 首次未配模型时，会弹出 **模型管理** 窗口  

### 4.3 对话示例

```
帮我看一下 src/main.rs 有没有明显 bug
把 README 里的安装说明改成中文
解释这个函数在干什么
运行测试并修复失败项
```

Agent 可能会：

1. 读文件、搜索代码  
2. 提议修改并写入  
3. 运行 shell 命令（视权限设置）  

请用账号 **有余额的 API Key**；错误信息常与 Key / 额度 / 网络有关。

## 5. 模型相关操作

| 场景 | 做法 |
|------|------|
| 切换当前模型 | `/model` 或 `Ctrl+M`，或 `/model deepseek-chat` |
| 新增 / 编辑 / 删除厂商 | `/model-add` 或 `/models` |
| 命令行改配置 | `yis models setup` / `yis models add ...` |
| 看当前有哪些模型 | `yis models` 或 TUI 里 `/model` |

配置保存在：`~/.yis/config.toml`（可备份，勿泄露 `api_key`）。

## 8. 配置文件位置

| 路径 | 内容 |
|------|------|
| `~/.yis/config.toml` | 模型、UI、功能开关等 **主配置** |
| `~/.yis/` | 会话记录、缓存等数据目录 |

备份模型配置：

```bash
cp ~/.yis/config.toml ~/yis-config.backup.toml
```

**不要**把含 `api_key` 的配置提交到公开 Git 仓库。

## 9. 权限与隐私边界

### 9.1 相对 xAI / grok.com（Release 默认）

- **不登录** grok.com / auth.x.ai  
- **不上报** 遥测、Sentry、会话到 xAI 云  
- **不拉取** 官方远程 settings / 模型目录 / 自动更新  

### 9.2 仍会出网（正常功能）

- 对话发往 **`~/.yis/config.toml` 里配置的 `base_url`**（如 DeepSeek）  
- 模型调用 `web_fetch`、联网搜索、MCP、shell 里的网络命令时  

完全离线：用本机 Ollama 等，并避免上述联网工具。

### 9.3 其它

- 工具可能读写当前项目文件、执行命令——请在信任的目录启动  
- API Key 存在本机 `config.toml`，勿提交到公开 Git

## 10. 常见问题 FAQ

### Q：`curl … install.sh` 报 404？

说明 GitHub **还没有成功的 Release**（或构建失败）。  
看 https://github.com/481617494/yis-Cli/releases 与 Actions。  
临时用源码：`cargo build -p xai-yis-pager-bin --release`。

### Q：启动后要登录 / 打开浏览器？

本发行版 **不需要** `yis login`。若仍提示登录，可能装的是官方 xAI 版，请确认：

```bash
which yis
yis models presets   # 本地版会列出厂商预设
```

### Q：数据会不会上传到 grok.com？

Release 默认 **不会** 向 grok.com / xAI 上传对话或遥测。  
对话会发给你配置的 **模型厂商**（见第 9 节隐私边界）。

### Q：对话失败 / 401 / Unauthorized？

1. `yis models` 看是否配置了模型  
2. 检查 Key 是否有效、账户是否有余额  
3. 检查 `base_url` 是否正确  
4. 用 `/model-add` 重新填写，或 `yis models setup`  

### Q：`yis: command not found`？

- 安装后把 `~/.local/bin` 加入 PATH  
- 或直接：`./target/release/yis`  

### Q：如何换模型 / 加第二个厂商？

```bash
yis models setup
# 或 TUI 内 /model-add
# 切换：/model
```

### Q：如何卸载？

```bash
rm -f ~/.local/bin/yis          # 或 Windows 下删除 yis.exe
# 可选：删除数据（会话与配置）
# rm -rf ~/.yis
```

### Q：界面怎么改成英文？

```
/language en
# 或启动前
YIS_LANG=en yis
```

### Q：`target` 很大能不能删？

能。`cargo clean` 或 `rm -rf target`，只影响编译缓存，不影响 `~/.yis` 配置。  
见 [本地开发与运维.md](./本地开发与运维.md) 第 7 节。

## 12. 推荐上手流程

```text
1. 安装 yis（curl 或本机编译）
2. yis models setup      ← 选厂商 + 填 Key
3. cd 到你的项目目录
4. yis
5. 输入需求，例如：「总结这个项目结构」
6. 用 /model 切换模型；用 /new 开新会话
```
